## How to Contribute

### Reporting Bugs
If you find a bug, please check the [Issues] tab to see if it has already been reported. If not, open a new issue with:
* **Description:** A clear summary of the problem.
* **Steps to Reproduce:** A list of actions that lead to the issue.
* **System Info:** Your OS (e.g., Windows 11) and the version of Qt/MinGW you are using.
* **Logs:** If the application crashes, include any error messages from the console or log files.

### Suggesting New Features
If you have a feature idea, please **open an issue first** to discuss it. We want to ensure that new features align with the project's goal of providing a lightweight, stable GUI wrapper for `karttapullautin`.

### Submitting Code
1. **Fork the repository.**
2. **Create a branch** for your feature or fix.
3. **Commit your changes** with clear, descriptive messages.
4. **Submit a Pull Request (PR):**
    * Explain why the change is necessary.
    * Ensure your code follows the existing style (C++17, consistent with `MainWindow` and `SettingsPanel` patterns).
    * If you are adding a new dependency (e.g., a new library), please explain why it is required.
